# Sync Patterns

## Decision Log

Each decision should include:

- Decision.
- Reason.
- Scope affected.
- Date.
- Decider.
- Follow-up task, if any.

## Action Item Quality

Weak:

- "確認する"
- "対応する"

Strong:

- "FE: /user-profile/config の表示条件を現行システムと比較し、差分をコメントに記載する"
- "BE: permission check source and affected roles, then report exact condition"

## Escalate When

- Spec and source behavior conflict.
- Customer wording can be interpreted in multiple ways.
- Release date depends on unconfirmed data or external approval.
- QA cannot create required data.
