# Spec Transfer Style

## Japanese Customer Docs

- Use natural Japanese appropriate for a non-technical customer.
- Prefer concise headings and table labels.
- Avoid internal phrases such as "testcase", "I used this source", or implementation scaffolding.
- For reports, `体言止め` is acceptable when the document already uses that style.

## Bilingual Transfer

- Japanese customer text: precise, natural, and business-facing.
- Vietnamese developer notes: direct, technical, and explicit about source/assumption.
- Keep code identifiers, route paths, DB names, product labels, and task IDs unchanged.

## Formatting

- Preserve heading hierarchy.
- Preserve bullet nesting when meaningful.
- Preserve links and attach them only to the intended token.
- Preserve table columns unless the user asks to redesign the table.
